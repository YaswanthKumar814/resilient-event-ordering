Lamport Clock

              

                A Lamport timestamp gives each event a logical time so the collector can rebuild
                a deterministic order even when physical arrival is delayed.
              

            

            

              
Vector Clock

              

                A vector clock tracks distributed causality. It helps decide whether one event
                happened before another or whether events are concurrent.
              

            

            

              
causal_before

              

                The previous event logically happened before the current event. This usually
                indicates a direct workflow dependency.
              

            

            

              
concurrent

              

                Neither event logically dominates the other. They are independent in the vector
                clock sense, even if one arrives later.
              

            

            

              
Per-Order Reconstruction

              

                The system reconstructs ordering separately for each order_id. It
                does not claim a single global order across unrelated customer orders.
